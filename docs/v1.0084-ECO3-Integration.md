## Как ECO-3 използва споделените настройки

ECO-3 получава конфигурацията от екосистемата по два начина:

1. **Frontend** (public/eco-3/):
   - Зарежда `/shared/js/config.js` → получава `KCY_CONFIG.urls.eco3`, `KCY_CONFIG.api.eco3`
   - Зарежда `/shared/js/navigation.js` → показва екосистемната навигация
   - Зарежда `/shared/css/common.css` → за навигацията
   - Чете `CRYPTO_CONFIG.PRICING.ECO3` → за ценообразуване
   - Чете `CRYPTO_CONFIG.TREASURY_WALLETS` → за крипто плащания
   - Получава Stripe publishable key от `/api/eco3/stripe-key`

2. **Backend** (private/eco-3/):
   - Чете **СЪЩИТЕ** Stripe ключове от `.env` (идентични с Chat .env)
   - Проксира Anthropic API (ключът е само на сървъра)
   - Nginx проксира `/api/eco3/` → `localhost:3001`

## Stripe интеграция

ECO-3 използва **СЪЩИТЕ Stripe ключове** като Chat:

```env
# В private/eco-3/configs/.env:
STRIPE_SECRET_KEY=sk_test_...    # СЪЩИЯТ като в Chat .env
STRIPE_PUBLISHABLE_KEY=pk_test_...  # СЪЩИЯТ като в Chat .env
```

## Стъпки след deploy

1. **Копирай пълния eco3.jsx** → `public/eco-3/app.jsx`
2. **Създай .env** → `cp private/eco-3/configs/.env.example private/eco-3/configs/.env`
3. **Добави ключове** → Stripe (същите), Anthropic (нов)
4. **Инсталирай зависимости** → `cd private/eco-3 && npm install`
5. **Стартирай** → `pm2 start server.js --name kcy-eco3`
