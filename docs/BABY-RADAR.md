# Pupikes Baby Radar (baby-monitor) — известия и модел на работа

Как приложението те информира, какви известия праща, и двуфоновият модел
„детегледачка ↔ майка тереза".

---

## 1. Как работи СЕГА (един телефон)

В момента приложението е **еднофоново**: телефонът, който гледа детето (с камерата),
**сам си показва известието**. Тоест същият телефон до детето вдига алармата.

Потокът (`src/core/monitor.js` → `raiseEvent`):
1. Камерата дава кадър → анализ за движение (всеки кадър) + хора (по-рядко, coco-ssd модел).
2. При събитие → записва се в дневника (`addEvent`) + **локално известие** на същия телефон.
3. По избор: best-effort POST към `relayUrl`, ако е зададен (виж раздел 3).

Всичко е on-device — нищо не се качва, освен ако сам не зададеш relay.

---

## 2. Какви известия праща детегледачката

Четири типа събития (`monitor.js`), всяко с дебоунс (да не спами):

| Тип | Кога се вдига | Текст | Настройка |
|-----|---------------|-------|-----------|
| **wake** (събуждане) | Устойчиво движение след спокойствие | „Детето се събуди (устойчиво движение)." | `alertWake` |
| **left** (излезе от кадър) | Не се вижда човек > 6 сек (след като е бил) | „Детето излезе от кадър." | `alertLeftFrame` |
| **stranger** (втори човек) | Преброени ≥ 2 души в кадъра | „Втори човек в стаята (непознат?)." | `alertStranger` |
| **fire** (ярък блясък) | Груба евристика за внезапен ярък блясък | „Възможен ярък блясък (ГРУБА евристика — НЕ е датчик за дим!)." | `alertFireHeuristic` |

> ⚠ „fire" е ГРУБА евристика по яркост, **НЕ** е датчик за дим. Не разчитай на нея за безопасност.

**Канал на известието** (`notifier.js`):
- Нативно (APK): `@capacitor/local-notifications` (системно известие).
- Браузър: Web Notification API.
- Звук според важността:
  - **Нормално** (wake, left) → мек двутонов сигнал.
  - **КРИТИЧНО** (stranger, fire) → настойчива **алармена мелодия** (тип сирена, ~2.4 сек,
    по-силна) + по-силна вибрация + заглавие с „⚠️" — за да личи веднага, че има проблем.
- Ако няма системно известие → toast + дневник в самото приложение.

Настройки за известията: `sound` (звук), `vibrate` (вибрация) + четирите `alert*` ключа.

## 3. Двуфонов модел: „детегледачка" ↔ „майка тереза"

**Целта:** два телефона с РАЗЛИЧНИ роли:
- 📷 **Pupikes Baby Radar** (monitor) — до детето, гледа с камерата, **НЕ** показва известия.
- 🔔 **Майка тереза** (родител) — до родителя, **получава** известията. Алармите излизат
  само тук.

**Защо трябва сървър (честно):** телефон не може да прати известие на ДРУГ телефон само
on-device. Нужен е канал (relay) между двата. Същият модел като при Selflearning Friend.

**Текущо състояние:** има ЗАГОТОВКА — `notifier.relay(relayUrl, payload)` праща събитието
по HTTP, а `monitor.js` го вика при `relayUrl`. Липсва: роли, сдвояване, и **приемната
страна** (родителският телефон да слуша и да вдига известието).

### Планиран дизайн (за изграждане)
1. **Роля** при пускане: „Pupikes Baby Radar" или „Майка тереза" (в настройки/onboarding).
2. **Сдвояване** с код за двойка (pair code) — двата телефона ползват един relay + един код.
3. **Relay сървър** (лек endpoint, като selflearning):
   - `POST /api/babymonitor/alert/<pair>` — детегледачката праща събитие.
   - `GET  /api/babymonitor/alert/<pair>` — родителят тегли новите събития (poll).
4. **Поведение по роля:**
   - Pupikes Baby Radar: вдига събития → праща към relay; локални известия по избор изключени.
   - Майка тереза: периодично пита relay → при ново събитие вдига известие (+ снимка по избор).
5. Без камера/контакти на родителския телефон — само получаване.

## 4. Файлове

| Файл | Роля |
|------|------|
| `src/core/monitor.js` | цикъл на наблюдение + `raiseEvent` (вдига събитията). |
| `src/core/notifier.js` | локални известия + звук/вибрация + `relay()` (към друг телефон). |
| `src/core/motion-detector.js` | движение / спи-будно / евристика за блясък. |
| `src/core/recognizer.js` | броене на хора (coco-ssd) → stranger / left. |
| `src/screens/config.js` | настройки (alert*, sound, vibrate, relayUrl). |
| `src/screens/dashboard.js` | живият изглед + дневник на събитията. |
