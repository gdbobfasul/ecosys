# Публикуване на приложения в RuStore (console.rustore.ru)

Този документ описва пълната процедура за подготовка и качване на нашите приложения в
RuStore (магазинът на VK за Русия). Допълва се и се изменя с всяко следващо публикувано
приложение. Огледален е на `docs/publish/huawei/how-to-publish.md` — разликите между двата
магазина са отбелязани изрично.

---

## Какво подготвяме за всяко приложение

Изходният код на RuStore изданието живее в `rustore/<име>/`. Материалите за магазина се
пазят в `rustore/<име>/publish/` (скрийншоти) и в общите генератори:

```
apk/<име>-rustore-release.apk        ← ПОДПИСАНИЯТ билд за качване (release-apks.sh)
rustore/<име>/publish/               ← скрийншоти (когато са отделни от huawei)
huawei/<име>/publish/store-listing/  ← описанията на 15-те езика (ОБЩИ за двата магазина)
huawei/<име>/publish/rustore-privacy.html ← политика за поверителност за RuStore (gen-privacy.mjs)
docs/store-console-texts.md          ← готови текстове за полетата на конзолата
```

### Пакетно име — суфикс `.rustore` (важи за ВСИЧКИ rustore апове)
`appId` в `rustore/<име>/capacitor.config.json` завършва на **`.rustore`**
(напр. `com.pupikes.newslator.rustore`, `com.pupikes.dodgemaster.rustore`). Така RuStore и
AppGallery изданията (`.hw`) никога не се сблъскват на едно устройство.
Пакетът се пипа САМО в `capacitor.config.json` — `android/` се регенерира при билд.

### Подписване
`deploy-scripts/release-apks.sh <име>` прави подписан release APK с ключ по приложение
(`keystores/<име>-release.jks` + `.cred`). **Ключът не се губи** — загубен ключ = невъзможен
ъпдейт. RuStore приема директно подписани APK (не изисква AAB).

### Локализация
RuStore изисква задължително **руско** описание; ние качваме и останалите от 15-те езика
на екосистемата, където формата позволява: `bg, ru, uk, en, de, fr, es, es-MX, it, pt, ar,
hi, ja, ky, zh-Hant`. Описанията се вадят от i18n на самото приложение (както за Huawei).

## Стъпка 0 — проверка (clearance) на името ПРЕДИ качване

Същото правило като при Huawei: името минава през name-check робота
(`node private/app-publisher-bot/app-publisher-bot.cjs name "<Име>"`) — App Store,
Google Play, AppGallery, търговски марки (TMview), домейни. Резултатът се пази в
`private/app-publisher-bot/name-checks/` и в `huawei/<ап>/publish/ANALYSIS.md`.
Пълната прясна таблица за всичките ни приложения: `ALL-APPS-NAME-TABLE.md` (пак там).

За изцяло НОВИ имена (търсим име без каквато и да е следа в интернет) се ползва
**ловецът на „призрачни" имена**: `node private/app-publisher-bot/ghost-names.cjs 5-10 30`
→ доклад `GHOST-NAMES-5-10.md`; актуалната таблица с анализа е в
`docs/publish/GHOST-NAMES.md`.

## Полета в конзолата на RuStore

| Поле | Какво попълваме |
|------|-----------------|
| Название | търговското име (виж `capacitor.config.json` → `appName`) |
| Пакет | `com.pupikes.<име>.rustore` — трябва да съвпада с подписания APK |
| Категория | по приложение (игра/новини/инструменти…) |
| Возрастная маркировка | по съдържанието (повечето наши: 12+) |
| Описание | от `store-listing/ru.txt` (+ останалите езици където може) |
| Скрийншоти | от `publish/` (портретни, реални екрани на апа) |
| Политика конфиденциальности | линкът към нашата политика (`https://<домейн>/privacy/<ап>.html`) |
| Контакт | dai.group.ltd.support@gmail.com |

Готовите текстове за копиране поле-по-поле са в `docs/store-console-texts.md`.

### Разлики спрямо Huawei AppGallery
- RuStore НЕ изисква `.hw`-подобен запазен суфикс — ползваме нашия `.rustore`.
- RuStore изисква руски език на листинга; Huawei приема направо мулти-език.
- Плащания: в RuStore е RuStore Billing (ние засега публикуваме безплатни апове/IAP по
  политиката в `publish/monetization.json`); никакво крипто (одитният ни ангажимент).
- Ъпдейт се качва със СЪЩИЯ подпис (ключът от `keystores/` — да не се сменя никога).

## Стъпки накратко (за всяко приложение)

1. `deploy-scripts/release-apks.sh <име>` → `apk/<име>-rustore-release.apk` (подписан).
2. Провери версията в апа (долу на началния екран) — тя трябва да расте при всеки ъпдейт.
3. Качи APK в конзолата, попълни полетата от `docs/store-console-texts.md`.
4. Прикачи скрийншотите и политиката за поверителност.
5. Изпрати за преглед; статусът се отразява в таблицата по-долу.
