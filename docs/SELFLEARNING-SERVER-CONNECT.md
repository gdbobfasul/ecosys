# Selflearning Friend — свързване на роботчето към сървъра

Как роботчето (телефон + десктоп) се свързва към своя relay-сървър. Има **два начина**:
**(А) с един таен ключ** — най-лесно, за нов/прехвърлен робот; **(Б) ръчно** — домейн + token.

---

## 1. Какво е сървърът (и какво НЕ е)

Сървърът е **лек relay** — услугата `kcy-selflearning` (node на порт `:3013`) зад nginx на
`/api/selflearning/`. Държи опашката „Слушай", knowledge snapshot и (по избор) канала за
команди — namespace-нати по **token**. Без акаунти, без контакти, без проследяване.

**Знанието по подразбиране живее ЛОКАЛНО на устройството.** Сървърът е канал за
синхронизация/„Слушай" (и по избор — сървърно съхранение). Правило: **1 сървър = 1 робот.**

---

## 2. Подготовка на сървъра (еднократно)

| Стъпка | Опция | Какво прави |
|-------|-------|-------------|
| 1 | **2** (или 4) | Качва проекта на сървъра (скриптовете се четат от `/var/www/deploy`). За ЧУЖД сървър избери таргет **custom** — приема всеки сървър, не само познатите. |
| 2 | **33** | Домейн + SSL за `selflearning.bot.nu` (или твоя домейн). Нужно е, за да се сервират `/api/...` и тайният файл по HTTPS. |
| 3 | **38** | Вдига самата услуга (systemd `kcy-selflearning` + nginx). Сървърът е готов да приема заявки от робота. |

> **38** настройва сървъра да приема заявките на робота (за който и да е сървър/домейн).
> **39** реално свързва робота (token + таен ключ). Опция 39 с „деплой" включва и работата на 38.

---

## 3. (А) Свързване с ЕДИН ТАЕН КЛЮЧ — най-лесно

Идеално за **нов или прехвърлен робот**: цялата конфигурация се записва във файл, който
апът тегли само по тайния ключ. В апа въвеждаш **само ключа**.

1. На сървъра пусни **опция 39**. Накрая те пита:
   ```
   Таен ключ [Enter = <генериран>]:
   ```
   Въведи произволен таен ключ (или Enter за генериран). Скриптът записва настройките на:
   ```
   https://selflearning.bot.nu/<ключ>/connection.bot.token
   ```

2. В апа: Настройки → „Източници на знание" → **🔑 Свържи с таен ключ** → въведи **само ключа**
   (или целия URL) → „Свържи с ключа". Апът тегли файла и се конфигурира сам.

### Какво има във `connection.bot.token`
Всичко нужно за връзка — нищо друго не въвеждаш:
```json
{
  "kind": "slf-connection",
  "version": 1,
  "domain": "selflearning.bot.nu",
  "token": "<owner-token>",
  "api": "/api/selflearning",
  "storage": "local",
  "features": { "sync": true, "listen": true, "exec": false },
  "createdAt": "…"
}
```

### ⏱ Сигурност: файлът се САМОУНИЩОЖАВА
- Тайният URL съдържа token-а, затова **изчезва автоматично след 10 минути** (детачнат
  таймер на сървъра — `systemd-run`, не блокира менюто), **или при следващо пускане на 39**.
- Свържи апа в този 10-минутен прозорец. После никой друг не може да ползва URL-а.
- Това трие само **публикувания файл** — **owner-token-ът остава** (роботът работи нормално).
- Ръчно по-рано: `sudo rm -rf /var/www/html/selflearning/<ключ>`.
- Бележка: файлът е в уеб-корена → ако деплойнеш СЛЕД 39, пусни 39 пак за нов ключ.

---

## 4. (Б) Ръчно свързване — домейн + token

Ако предпочиташ: опция 39 показва накрая и **домейна + token-а**. В апа →
„🔗 Свържи към сървър (ръчно)" → въведи двете полета → „Запази връзката".
Апът сам сглобява пълните адреси по каноничната схема.

---

## 5. Канонична схема на адресите

Апът въвежда само домейн + token (или таен ключ); пълните адреси ги сглобява сам:

```
base   = https://<домейн>/api/selflearning
sync   = <base>/sync/<token>      (износ на знание към сървъра)
listen = <base>/listen/<token>    (режим „Слушай")
exec   = <base>/exec/<token>      (команди на машини — OPT-IN, виж SELFLEARNING-REMOTE-EXEC.md)
health = <base>/health            (проверка на живо — без token)
```

- Апът: `core/server-link.js` (`buildUrls`, `connectWithKey`).
- Сървърът: `deploy-scripts/server/23-link-selflearning-robot.sh` + `private/selflearning-server/server.js`.

---

## 6. Къде се пази token-ът

| Място | Път / ключ | Бележки |
|-------|------------|---------|
| **Сървър** (източникът) | `/var/lib/kcy-selflearning/data/owner-token` | Права `600`. Извън деплой пътя → никой деплой не го трие. Нов само при РЕСЕТ. |
| **Сървър** (таен файл, временен) | `/var/www/html/selflearning/<ключ>/connection.bot.token` | Публикува се от 39; **самоунищожава се след 10 мин** / при следващо 39. |
| **Апът** (твоето копие) | състояние под ключ `slf.state.v1` → `settings.server = { domain, token, storage }` | APK: Capacitor Preferences. Браузър/кеш: localStorage. |

---

## 7. Съхранение на знанието — локално или сървър

- **Локално (по подразбиране)** — знанието живее на устройството (Preferences/localStorage),
  работи офлайн, локалното е винаги главно (master).
- **Сървър (по избор)** — за бързо растящо знание. Полето `storage:"server"` в
  `connection.bot.token` казва на апа да ползва сървърно съхранение. *(Сървърният backend
  за мащабно съхранение — PostgreSQL — е отделна, по-голяма стъпка; виж бележките по проекта.)*

---

## 8. ⚠ Базата данни — кой я трие и кой НЕ

- Базата на робота е в **`/var/lib/kcy-selflearning/data`** — извън деплой пътя. **Никой деплой
  не я пипа.**
- Трие се **САМО** от опция 39 → РЕСЕТ (`--reset`). Ъпдейт на домейн/деплой **НЕ** я трие и
  **НЕ** сменя token-а.

---

## 9. Проверка, че работи

- Отвори `https://selflearning.bot.nu/` — landing страницата проверява `/api/selflearning/health`.
- Ако „не отговаря" → пусни опция **38** (или 39 с деплой).
- В апа след свързване режимът „Слушай" и синхронизацията тръгват сами.
