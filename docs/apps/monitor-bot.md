# Pupikes Site Monitor (Source-Watch Bot) — как функционира

Самостоятелно мобилно приложение от типа „робот под наем": активираш робот, който следи **източник** (RSS/Atom емисия или публично JSON API) и те известява **локално**, когато се появи нов запис или съвпадне ключова дума. Без акаунт, без облак, без проследяване, без реклами, без ключове. Издава се за **RUStore** и **Huawei AppGallery** като две идентични, независими приложения (различават се само по app id и акцент на иконата).

- **App ID:** `com.kcy.monitorbot.rustore` (RUStore) / `com.kcy.monitorbot.huawei` (Huawei)
- **Стек:** Vanilla JS + Vite 5 + Capacitor 6

## Поток (наеми → настрой → разреши → следи)
1. **Наеми робота (онбординг)** — обяснение какво прави роботът + бутон „Активирай робота". Безплатно ядро; „Премиум наем" е STUB през магазина (нищо не таксува).
2. **Настройка на монитор** — добавяш монитори. Всеки има: източник (RSS/Atom URL или JSON API URL), правило (нов запис и/или ключови думи), честота (15 мин / 1 ч / дневно). Има готови безплатни пресети за бърз тест.
3. **Разрешения** — интернет (за теглене на източниците) + известия, с ясни обяснения.
4. **Табло** — главен ключ ON/OFF, списък монитори с последна проверка/съвпадение, дневник, редакция/пауза/триене и поле за CORS прокси.

## Как работи проверката, дифът и известието
- **Fetch** (`src/core/fetcher.js`) — тегли източника. RSS/Atom се разбира през `DOMParser` (без тежки зависимости); JSON се чете с `JSON.parse` и извличане на списък по път с полета id/заглавие.
- **Diff** (`src/core/differ.js`) — пази локално id-тата на видените записи в самия монитор (`seenIds`). Първата проверка само „заучава" (без известие). После сравнява и намира НОВИ и/или съвпадащи по ключова дума записи.
- **Notify** (`src/core/notifier.js`) — на устройство през `@capacitor/local-notifications`; в браузър през Web Notifications (ако са разрешени), иначе тих лог.
- **Scheduler** (`src/core/scheduler.js`) — всеки монитор е „падежен" по своя интервал. На устройство и в уеб работи същата `tick()` логика; при отваряне веднага се прави един тик.

## Фонов режим (честно)
Уеб таймерът и тикът при отваряне покриват „проверка докато приложението е отворено" и „наваксване при отваряне". За ИСТИНСКИ периодичен фон (приложението затворено) е нужен background-runner плъгин + системни разрешения за фонова работа. Това е документирано, не е фалшиво симулирано.

## CORS (честно)
Произволни сайтове често **блокират** cross-origin fetch от браузър (CORS). Източници с включен CORS работят директно (много публични JSON API и някои RSS). За произволни сайтове можеш да зададеш СВОЙ безплатен CORS/RSS-to-JSON прокси в Табло → „CORS прокси" (по подразбиране празно). Не се хардкодва платена услуга, а CORS грешките се показват четимо.

## Управление на данните
- Конфигурацията на мониторите и id-тата на видените записи се пазят **само локално** през `@capacitor/preferences`.
- Нищо не се качва в облак; няма акаунт. Всеки монитор и цялата конфигурация се трият от потребителя.

## Разрешения
- **INTERNET** — за теглене само на адресите, които потребителят сам е въвел.
- **POST_NOTIFICATIONS** — локални известия при нов запис/съвпадение (незадължително).

## Езици
Интерфейсът е на **15-те езика на екосистемата** (bg, ru, uk, en, de, fr, es, es-MX, it, pt, ar, hi, ja, ky, zh-Hant). Език по подразбиране: **руски**.
