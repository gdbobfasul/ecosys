# Pupikes FAQ Desk — как функционира

Self-contained приложение „robot for rent": малък бизнес **наема/активира** робот, който отговаря автоматично на честите въпроси на клиентите **по правила (ключови думи)** — **без платен LLM, без акаунти, без контакти, без проследяване**. Vite + Capacitor 6, без Google Mobile Services. App ID: `com.pupikes.businessfaqbot.rustore` / `com.pupikes.businessfaqbot.huawei`.

## Поток на работа
**Активирай робота → Разрешения → База знания → Демо чат.**

## Екрани
- **Онбординг** (`screens/onboarding.js`) — „наеми робота"; Premium е STUB (no-op).
- **Разрешения** (`screens/permissions.js`) — само известия; явно описва какво НЕ прави.
- **База знания** (`screens/kb-config.js`) — Q&A CRUD + поздрав/резервен отговор/ескалация + бързи бутони + работно време.
- **Канали** (`screens/channels.js`) — честен статус на месинджърите (WhatsApp/Viber/Messenger показани като „pending native").
- **Табло** (`screens/dashboard.js`) — ON/OFF, тест конзола, дневник, броячи.
- **Демо чат** (`screens/demo-chat.js`) — вграден чат за тест на правилата.

## Как работи (ядро)
- **Rule engine** (`core/rule-engine.js`) — входът се нормализира (lowercase, без диакритика/пунктуация); за всеки активен Q&A запис се броят съвпаденията на ключови фрази, претеглени по дължина (по-дълга фраза = по-специфично → по-висок приоритет). Печели записът с най-висок резултат; при нула съвпадения → резервен отговор + ескалация.
- **Работно време** (`core/office-hours.js`) — 24/7 или интервал (вкл. през полунощ); извън него → away съобщение.
- **respond.js** — комбинира rule-engine + office-hours + лог + броячи.
- **channel-adapter.js** — канал `local` (работи) + whatsapp/viber/messenger STUB-ове.
- **notifier.js** — `@capacitor/local-notifications` / toast fallback. **storage.js** — localStorage (seed база знания), без мрежа.

## Реалност на месинджърите (честно)
- **Работи сега:** канал `local` (демо чат) — изцяло on-device, тестваем в браузъра.
- **Изисква native:** WhatsApp/Viber/Messenger през `NotificationListenerService` + direct-reply и системно разрешение „Notification access" (native Kotlin плъгин — план в `native-plugin/`). Статусът е „pending native"; изпращането **не** се фалшифицира.
- **Извън обхвата:** официалните сървърни bot API-та.

## Къде се пазят данните
Базата знания и дневникът се пазят **само локално** (localStorage). Без контакти, без акаунти, без локация, без проследяване. Потребителят може да трие/нулира записите.

## Разрешения
- **Известия** (локални) — за нови съобщения/ескалация; незадължително.
- **INTERNET** — стандартно за Capacitor WebView обвивката; приложението не прави мрежови заявки.

## Езици
Интерфейсът **не** ползва многоезичния слой на екосистемата — няма i18n модул; текстовете са фиксирани (предимно на български). UI-ят засега е едноезичен; магазинните листинги се водят на 15-те езика.
