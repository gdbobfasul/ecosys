4. Такси: 0.08% (много ниски!)

Структура:

Такса Процент Към къде
Burn 0.03% Изгаряне (намалява totalSupply)
Owner 0.05% Owner адрес
Общо 0.08% -

Формула:

FEE_DENOMINATOR = 100,000 BURN_FEE = 30 // 30/100,000 = 0.03% OWNER_FEE = 50 // 50/100,000 = 0.05% burnAmount = (amount * 30) / 100,000 ownerAmount = (amount * 50) / 100,000 transferAmount = amount - burnAmount - ownerAmount

Пример с 1000 токена:

Транзакция: 1000 KCY1
  • Burn: 0.3 токена (унищожени)
  • Owner: 0.5 токена
  • Получател: 999.2 токена
⚠️ Кога се прилагат таксите:
  • ✅ Когато поне ЕДНА страна е normal потребител
  • ❌ НЕ се прилагат когато И ДВЕТЕ страни са exempt

✅ СЦЕНАРИЙ 1: Exempt ↔ Exempt

Когато И ДВЕТЕ страни са exempt:

6. Trading Lock (48 часа)

За Normal потребители:

  • ❌ 48 часа след deployment - НЕ могат да търгуват
  • ✅ След изтичане - пълна свобода (в рамките на лимитите)

За Exempt адреси:

  • ✅ Могат веднага след deployment
  • ✅ Bypass на trading lock
// Проверка дали trading е активен bool enabled = isTradingEnabled(); // Колко време остава (в секунди) uint256 remaining = timeUntilTradingEnabled(); // Trading се активира автоматично след 48ч // tradingEnabledTime = deploymentTime + 48 hours

7. Cooldown система (2 типа)

1. Normal user cooldown (2 часа):

COOLDOWN_PERIOD = 2 hours mapping(address => uint256) public lastTransactionTime;
  • Записва се при всяка normal user транзакция
  • Проверка: block.timestamp >= lastTx + 2 hours
  • Прилага се само когато normal изпраща

2. Exempt slot→Normal cooldown (24 часа):

EXEMPT_TO_NORMAL_COOLDOWN = 24 hours mapping(address => uint256) public lastExemptToNormalTime;
  • Записва се само при exempt slot (1-4) → normal
  • Проверка: block.timestamp >= lastExemptTx + 24 hours
  • НЕ се прилага за Router/Factory/Owner

Exempt↔Exempt:

✅ БЕЗ cooldown (не се записва нищо)

8. Liquidity Pair управление

Идея:

  • ❌ Normal users НЕ могат да добавят/премахват ликвидност директно
  • ✅ Normal users МОГАТ да купуват/продават през Router
  • ✅ Exempt addresses могат всичко

Blocking логика:

// Блокира добавяне на ликвидност if (!fromExempt && isLiquidityPair[to]) { revert("Normal users cannot add liquidity directly"); } // Блокира премахване на ликвидност if (!toExempt && isLiquidityPair[from] && msg.sender != Router) { revert("Normal users cannot remove liquidity directly"); }

Управление на Pair адреси (Owner функции):

Стъпка 1: Прочети Pair адреса

// VIEW функция - не модифицира контракта getLiquidityPairAddress(WBNB_address) // Връща: Pair адреса от Factory // Връща: address(0) ако pair не съществува

Стъпка 2: Добави ръчно Pair адреса

// За един pair setLiquidityPair(pair_address, true) // За множество pairs setLiquidityPairBatch([pair1, pair2, pair3], true)

Стъпка 3: Заключи завинаги

lockLiquidityPairsForever() // ❌ След това НЕ могат да се добавят/махат pairs
⚠️ Важно:
  • Ликвидността трябва да е добавена ПРЕДИ да извикаш функцията
  • Pair адресите са постоянни за всяка двойка токени
  • Един trading pair = Един постоянен Pair адрес

9. Pause механизъм

Активиране (само Owner):

pause() // Спира контракта за 48 часа PAUSE_DURATION = 48 hours

Ефект:

Тип адрес Може ли да прави транзакции?
Normal users ❌ НЕ
Exempt addresses ✅ ДА (игнорират pause)

Проверки:

// Дали е на пауза bool paused = isPaused(); // Колко време остава (в секунди) uint256 remaining = timeUntilUnpaused(); // Пауза може да се активира многократно // Всеки път: +48 часа от момента на извикване
