1. Общ преглед

KCY1 Website е изцяло интегриран с blockchain чрез Web3/Ethers.js библиотеките. Всички данни се зареждат директно от smart contract-а, без нужда от ръчно въвеждане.

✅ Какво работи автоматично:

  • 🕐 48-часов таймер - чете tradingEnabledTime от контракта
  • 📊 Статистики - total supply, burned tokens, circulating supply
  • 🚫 Блоклист - всички блокирани адреси от contract events
  • 💰 Market cap - автоматично изчисление
⚠️ Важно: Сайтът НЕ работи докато не конфигурираш contract адреса след deployment!

2. Как работи интеграцията

🔗 Web3 Stack:

Компонент Версия Предназначение
Ethers.js 5.7.2 Комуникация с blockchain
BSC RPC Mainnet Binance dataseed node
Contract ABI Minimal Само нужните функции

📡 RPC Endpoint:

https://bsc-dataseed.binance.org/
ℹ️ Информация: Използваме публичния Binance RPC node. Безплатен е, но има rate limits. За production може да използваш платен RPC (QuickNode, Ankr).

🎯 Как се извличат данните:

Стъпка 1
Инициализация при зареждане на страницата:
// JavaScript се изпълнява при отваряне window.addEventListener('DOMContentLoaded', async () => { await initWeb3(); // Свързва се с BSC updateCountdown(); // Стартира таймера });
Стъпка 2
Свързване към контракта:
provider = new ethers.providers.JsonRpcProvider(rpcUrl); contract = new ethers.Contract(contractAddress, ABI, provider);
Стъпка 3
Извличане на данни:
// Пример: Total Supply const totalSupply = await contract.totalSupply(); // Пример: Trading Enabled Time const tradingTime = await contract.tradingEnabledTime(); // Пример: Blacklist Events const events = await contract.queryFilter(filter);

3. Стъпки след deployment на токена

Стъпка 1

Deploy Smart Contract на BSC Mainnet

  1. Deploy kcy-meme-1.sol на BSC Mainnet
  2. Запази contract адреса (например: 0xABC123...)
  3. Verify contract на BSCScan
Стъпка 2

Отвори index.html файла

  1. Отвори index.html с текстов редактор (VS Code, Notepad++)
  2. Намери секцията CONFIGURATION (около ред 350)
Стъпка 3

Конфигурирай contract адреса

Намери това:

const CONFIG = { contractAddress: '0xYOUR_CONTRACT_ADDRESS_HERE', // ← ПРОМЕНИ ТОВА! rpcUrl: 'https://bsc-dataseed.binance.org/', bscScanUrl: 'https://bscscan.com', pancakeswapUrl: 'https://pancakeswap.finance/swap', chatServerUrl: '', chatAppUrl: '', };

Промени на:

const CONFIG = { contractAddress: '0xABC123YOUR_REAL_CONTRACT_ADDRESS', // ✅ Реален адрес rpcUrl: 'https://bsc-dataseed.binance.org/', bscScanUrl: 'https://bscscan.com', pancakeswapUrl: 'https://pancakeswap.finance/swap', chatServerUrl: '', // По-късно добави chatAppUrl: '', // По-късно добави };
Стъпка 4

Запази и тествай

  1. Запази файла (Ctrl + S)
  2. Отвори index.html в браузър
  3. Отвори Developer Console (F12)
  4. Виж дали има грешки
✅ Ако всичко работи, ще видиш:
🚀 Initializing KCY1 Website... ✅ Web3 initialized successfully ✅ Connected to blockchain
❌ Ако нещо не работи, ще видиш:
⚠️ Contract address not configured! ❌ Web3 initialization failed: ...

5. Функционалност (какво показва сайтът)

⏰ 48-часов таймер:

  • Преди deployment: Таймерът е скрит (няма contract)
  • След deployment (първите 48ч): Показва оставащо време в реално време
  • След 48 часа: Таймерът автоматично изчезва
Как работи:
tradingEnabledTime = deploymentTime + 48 hours
Чете се директно от contract.tradingEnabledTime()

📊 Статистики:

Показател Откъде се взима
Market Cap Circulating Supply × Token Price
Token Price От liquidity pair (requires setup)
Circulating Supply totalSupply - contractBalance
Burned Tokens 1,000,000 - totalSupply
Liquidity От pair contract (requires setup)
Holders External API (BSCScan/indexer)
⚠️ Забележка: Някои статистики като Holders и 24h Volume изискват external API интеграция, която не е включена в базовата версия.

🚫 Блоклист:

  • Извлича всички Blacklisted events от контракта
  • Обработва ги и показва само текущо блокираните адреси
  • Автоматично се обновява при всяко отваряне на страницата
// Как работи: const filter = contract.filters.Blacklisted(); const events = await contract.queryFilter(filter, 0, 'latest'); // Обработка: // event.args.status === true → блокиран // event.args.status === false → разблокиран

6. Решаване на проблеми

❌ Проблем: "Contract address not configured"

Причина: Не си променил contractAddress в CONFIG
Решение: Промени 0xYOUR_CONTRACT_ADDRESS_HERE на реалния адрес

❌ Проблем: "Web3 initialization failed"

Възможни причини:
  • Няма интернет връзка
  • BSC RPC node не работи
  • Contract адресът е грешен
Решение: Провери console-а за точната грешка

❌ Проблем: Таймерът не се показва

Възможни причини:
  • Contract адресът не е конфигуриран → Нормално е
  • Вече са минали 48 часа → Нормално е, таймерът се скрива
  • Грешка при четене на tradingEnabledTime → Виж console

❌ Проблем: Статистиките показват "N/A"

Причина: Някои данни изискват допълнителна интеграция:
  • Token Price - трябва да се query-не liquidity pair
  • Holders - трябва external API (BSCScan)
  • 24h Volume - трябва external API
Решение: Тези са по-сложни интеграции и не са задължителни за старт

✅ Проверка дали всичко работи:

  1. Отвори сайта
  2. Натисни F12 (Developer Tools)
  3. Отвори таб Console
  4. Търси за съобщения:
    • ✅ "Web3 initialized successfully"
    • ✅ "Connected to blockchain"
  5. Отиди на таб Network
  6. Виж дали има requests към BSC RPC
