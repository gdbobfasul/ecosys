# KCY Тест Робот

Автоматичен робот, който обхожда сайта, тества за грешки/бъгове и събира лог + детайли
как са се случили — за лесен дебъг. Изграждаме го по фази.

## Какво прави (Фаза 1 — MVP)

1. Отваря **критичните пътища** на всяко приложение (chat / eco-3 / portals / HLB / WNB) с Playwright.
2. Лови клиентските грешки: **console грешки, необработени изключения, провалени заявки, HTTP 4xx/5xx**.
3. Пинга **health** endpoint-ите (`/api/health`, `/api/eco3/health`, …).
4. **Корелира със сървъра** — дърпа `/last-errors-bundle` (kcy-diag работи дори при 502) и вади подозрителните редове.
5. Пише **репорт** (`report.md` + `report.json`) и **екранна снимка** при грешка.

Източникът на адреси е `/tree` (`public/tree/tree.json`) — авто-генерираното дърво на екосистемата.

## Безопасност

- **prod** (`https://alsec.strangled.net`) — само **ЧЕТЕНЕ** (без форми/писане).
- **vm** (`https://192.168.0.108`) — тук в по-късна фаза ще пуснем **fuzz/random** (разрушителни сценарии). Самоподписаният TLS се игнорира.

## Първоначална настройка

```bash
cd private/robot
npm install
npx playwright install chromium      # сваля браузъра (еднократно)
```

## Пускане

```bash
node run.js                          # prod, критични пътища (read-only)
node run.js --target vm              # срещу VM
node run.js --all                    # ВСИЧКИ публични страници от дървото
node run.js --crawl                  # Фаза 2: crawler (следва линковете, BFS)
node run.js --crawl --max 200 --depth 4   # по-дълбоко
node run.js --target vm --fuzz       # Фаза 3: fuzz форми (САМО VM!)
node run.js --target vm --fuzz --seed 12345   # повтори същата последователност
node run.js --include-admin          # включи и админ страниците
node run.js --app portals            # само едно приложение (критични)
node run.js --headed                 # с видим браузър
```

Репортите се пишат в `private/robot/reports/<време>/`.

## Режими (фази)

| Режим | Флаг | Цел | Какво прави |
|---|---|---|---|
| Критични пътища | (по подразбиране) | prod/vm | по 1 сценарий на приложение + health |
| Пълно (дървото) | `--all` | prod/vm | всички публични страници от `tree.json` |
| **Crawler** | `--crawl` | prod/vm | следва откритите линкове (BFS), отбелязва форми |
| **Fuzz** | `--fuzz` | **само VM** | попълва и изпраща форми с гранични/зловредни стойности; `--seed` за повторение |

## Пускане от сайта (админ)

Хедър меню (само админ IP) → **🤖 Робот** → `/shared/robot.html`. Избираш цел/режим, натискаш ▶,
гледаш на живо, разглеждаш миналите репорти. Това върви през kcy-diag на сървъра — нужна е
инсталация веднъж: **меню точка 53** (или `sudo 32-setup-robot.sh`).

## Дървото на екосистемата (`/tree`)

```bash
node tree-gen.js             # регенерира public/tree/{index.html,tree.json}
```

Локален преглед: отвори `public/tree/index.html`. На сървъра: `https://alsec.strangled.net/tree`.
Регенерира се автоматично при деплой на сорса (меню точка 5).

## Следващи фази (план)

- **Фаза 2:** crawler — авто-обхождане на всички линкове/форми от дървото.
- **Фаза 3:** seeded fuzz срещу VM (произволни, но възпроизводими сценарии със записан seed).
- **Фаза 4:** пускане от менюто/cron, репорти до `/last-errors`, дашборд.
