# token-protector

Наблюдава токена on-chain и **реагира** при зловредни модели.

**Политика (по искане):** 🔔 аларма **винаги** · 🛑 авто-**pause** по избор · ❌ **БЕЗ** авто-blacklist
(блокирането на адреси е само аларма — **човек решава** кого да блокира ръчно).

## Какво следи
| Детектор | Реакция |
|---|---|
| **Източване/спад на ликвидност** (резервите падат рязко) | аларма + **кандидат за авто-pause** |
| **Голяма продажба** (dump ≥ N% от пула) | аларма; при ≥2× прага → авто-pause |
| **Голям приток към 1 адрес** (концентрация/кит) | само аларма |
| **Висока честота** (бот: ≥N трансфера/мин) | само аларма (ръчен blacklist решение) |

## Сигурност (ключово)
- Авто-pause се вика със **СПЕЦИАЛЕН guardian ключ** (`TOKEN_GUARDIAN_KEY` в `.env`) — **pauser, НЕ owner**.
- Owner-ът (mint / теглене / смяна на адреси) стои в **студен/multisig** ключ — никога тук.
- Ако няма guardian ключ → протекторът работи **само като аларма** (не може да паузира).

## Настройка (env)
```
TOKEN_CONTRACT_ADDRESS=0x...     # адресът на токена (от token-creator)
TP_NETWORK=bscTestnet            # testnet първо
TOKEN_GUARDIAN_KEY=0x...         # САМО ако искаш авто-pause (pauser ключ)
TP_AUTOPAUSE=true                # включи авто-pause (по подразбиране false)
# прагове (по избор): TP_LIQ_DROP, TP_DUMP, TP_WHALE, TP_BOT_RATE, TP_PAUSE_LIQ, TP_PAUSE_DUMP
TP_WEBHOOK=https://...           # по избор: Telegram/Discord известия
TP_ALLOWLIST=0xowner,0xdistrib   # адреси за пренебрегване
```

## Пускане
```bash
npm install
node protector.js --once         # една проверка (тест)
node protector.js                # непрекъснато наблюдение (Ctrl+C спира)
TP_AUTOPAUSE=true node protector.js   # с авто-pause (нужен guardian ключ)
```
Аларми се пишат в `alerts/protector.log` (+ конзола + webhook).

## Какво НЕ прави / граници
- Реактивен е — не спира **единична** вече-минала tx, но спира **по-нататъшната** щета (авто-pause).
- Най-силната защита е **в контракта** (max tx/wallet лимити, trading delay) — те пречат на лошите tx-и изобщо.
- Възможни фалшиви аларми (легитимен кит vs бот) → прагове + allowlist + степенувана реакция.
- **Първо срещу testnet токена** (от token-creator). После mainnet с реален guardian.
