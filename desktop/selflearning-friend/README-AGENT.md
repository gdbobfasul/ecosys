# 🛠 Агент — десктоп суперсили (само Electron build)

Този десктоп клонинг на „Pupikes Learning Buddy" има **агентски суперсили**, които
**не съществуват** в телефонните приложения (rustore/huawei). Те живеят в Electron
main процеса (Node) и са изложени към renderer-а през тясен, типизиран мост.

## Какво може агентът

1. **Файлова система** — четене/писане/листване **само в рамките на папка, която ти
   избираш** (sandbox). Пътища извън папката се отказват.
2. **Изпълнение на команди (shell)** — пуска команда (curl, build и т.н.), връща
   stdout/stderr/exit код.
3. **SSH** — свързва се към машина (Linux/Windows), която ти зададеш (хост/порт/
   потребител + ключ ИЛИ парола), изпълнява команда, връща изхода.
4. **Playwright (уеб ровене)** — отваря URL headless, извлича заглавие/текст/линкове,
   може да кара малки скриптове от действия (click/fill/waitFor) върху уеб-апове.

## ⚠ Честно предупреждение за риск

Тези мощи **могат да четат, пишат и трият файлове** и **да пускат команди** на този
компютър и на отдалечени машини през SSH. **Единствената защита е кодовата дума
(разковничето) + изричното потвърждение.** Ползвай ги внимателно — грешна команда може
да навреди. Не давай достъп до папка с важни данни, ако не си сигурен.

## Как е гейтнато ВСЯКО действие

Преди ВСЯКО мощно действие (запис на файл, shell, SSH, web crawl) изскача модал
**„⚠ ОПАСНА команда — потвърди"**, който:

- изисква **отключена сесия** (`isUnlocked()` от identity.js),
- показва **ТОЧНО какво ще се изпълни** (командата/пътя/URL дословно),
- изисква да **въведеш кодовата дума**; тя се сравнява с `nameHash` през
  `verifyCodeWord()` (същата логика като гейтнатите команди в commands.js).

Без вярната дума + натиснат „Изпълни" **нищо не тръгва**. (Листване/четене на файл и
четенето на хостове не са разрушителни и не искат дума; всичко, което **променя** нещо
или изпълнява команда — иска.)

## Модел на сигурност

- **Локални креденшъли:** SSH ключове/пароли и избраната базова папка се пазят само в
  `agent-config.json` в `userData` на приложението — на този компютър, без мрежа, без
  синхронизация.
- **Тесен preload:** `window.SLF_AGENT` излага само конкретни, именувани IPC методи —
  **никакъв суров `require`/`child_process`/`fs`** към renderer-а.
- **Гейтът е в renderer-а** (държи разковничето); main процесът само изпълнява
  поисканото. Самият main НЕ съдържа кодовата дума.
- **Телефонът е непокътнат:** rustore/huawei нямат този preload и нямат SLF_AGENT;
  екранът „🛠 Агент" се показва само когато `window.SLF_DESKTOP === true`.

## Playwright — браузър

Playwright е добавен **без** да сваля собствения си chromium (`PLAYWRIGHT_SKIP_BROWSER_
DOWNLOAD=1` при install). Handler-ът `agent:web:crawl` опитва в следния ред:

1. свален chromium на Playwright (ако има),
2. `PLAYWRIGHT_CHROMIUM_PATH` env (ако е зададен),
3. **системен Edge (`msedge`) или Chrome (`chrome`) канал** — обикновено наличен на
   Windows и **проверено работи на тази машина**.

Ако искаш собствен chromium на Playwright:

```
npx playwright install chromium
```

(Има и chromium в `private/robot/` на repo-то — може да насочиш Playwright към него
през `PLAYWRIGHT_CHROMIUM_PATH`, ако предпочиташ.)

## Файлове

- `electron/agent.cjs` — IPC handlers (FS/shell/SSH/Playwright) в main процеса.
- `electron/preload.cjs` — тесен мост `window.SLF_AGENT`.
- `electron/main.cjs` — регистрира handler-ите при `app.whenReady()`.
- `renderer/src/screens/agent.js` — екранът „🛠 Агент" (само на десктоп).
- `renderer/src/ui/confirm-gate.js` — модалът „⚠ ОПАСНА команда" с кодова дума.
- `renderer/src/core/identity.js` — `verifyCodeWord()` за гейта.

## TODO / честни ограничения

- За Playwright върху системи **без** Edge/Chrome и **без** свален chromium —
  трябва `npx playwright install chromium` (тежко сваляне).
- SSH е тестван само на ниво зареждане на библиотеката; за реален тест трябва жив хост
  (host/user + ключ/парола), който собственикът задава.
- Размерът на .exe нараства заради playwright (~+няколко MB код, без браузър).
