# Тестване — Самообучаващ се приятел (RUStore)

## Браузър (бързо)
1. `npm install`
2. `npm run dev` → отвори показания localhost адрес.

## Сценарии за приемане

### 1. Раждане → име
- При първо отваряне се вижда дружелюбно лице и въпрос „Как да се нарека?“.
- Въведи име два пъти (напр. „Пешо“), потвърди. Очаквано: преминава директно в чата (вече си отключен).

### 2. Единичен потребител — заключване
- Настройки → намали „Неактивност преди заключване“ на 1 минута, или натисни „🔒 Заключи сега“.
- Презареди / изчакай. Очаквано: екран „Как се казвам?“.
- Въведи грешно име → дружелюбен отказ + намаляващ брояч опити.
- След изчерпване на опитите → кратка пауза (cooldown) с обратен брояч.
- Въведи правилното име → влизаш в чата.
- Бутон „Дай ми подсказка“ показва първа буква + брой букви (без да издава името).

### 3. Самообучение — Q&A
- В чата напиши: `като кажа добро утро, отговаряй Слънце мое!`
- Очаквано: „Разбрах! Като кажеш…“.
- Напиши `добро утро` → отговор „Слънце мое!“ (таг „от паметта“).

### 4. Самообучение — факт
- Напиши: `запомни, че любимият ми цвят е син`
- Очаквано: „Запомних: …“. Виж го в екран „Памет“.

### 5. Корекция
- След отговор от паметта напиши: `не, всъщност любимият ми цвят е зелен`
- Очаквано: „Поправих се…“. В „Памет“ стойността е обновена.

### 6. Памет (CRUD)
- Екран „Памет“: добави ръчно, редактирай съществуващ, изтрий. Всичко се запазва след презареждане.

### 7. AI helper + fallback
- Настройки: остави „Безплатен AI помощник“ включен, онлайн → отвори въпрос извън паметта; може да дойде по-богат отговор (таг „AI“).
- Изключи мрежата или изключи helper-а → отговорите идват от памет/правила, без грешки (graceful fallback).

### 8. Нулиране
- Настройки → „Изтрий всичко и забрави ме“ → потвърди. Връща се към екрана за раждане.

## Бележки
- Без Android SDK APK не се строи тук (оркестраторът поема APK).
- `npm run build` трябва да минава чисто; `node --check` на всички src файлове — без грешки.
