# RUStore — тестова среда / затворено тестване

## 1. Регистрация на разработчик
1. Влез в RUStore Console: https://console.rustore.ru
2. Потвърди акаунт на юридическо лице/самозаето (изисква се за Русия).
3. Създай ново приложение с package `com.pupikes.planeshooter.rustore`.

## 2. Качване на тестов APK/AAB
- RUStore приема **APK** и **AAB**. За тестване качи `app-debug.apk` или подписан
  `app-release.apk`.
- Качването става в раздела **„Версии"** на приложението.

## 3. Затворено тестване (sandbox)
RUStore предлага **„Тестване"** със списък от тестери по e-mail/акаунт:
1. В конзолата → приложение → **„Тестване" (Closed testing)**.
2. Добави e-mail адресите на тестерите (RUStore акаунти).
3. Качи тестовата версия в тестовия трек.
4. Тестерите получават линк/виждат приложението в RUStore приложението си.

## 4. Билинг sandbox (само ако се добави IAP — НЕ е активно тук)
- RUStore Billing SDK има **тестов режим**: покупките не теглят реални пари.
- Тестовите акаунти трябва да са добавени към списъка с тестери.
- Тъй като текущата версия НЯМА покупки, този раздел е TODO (виж README → RUStore SDK).

## 5. Ревю преди публикация
- Времето за ревю обикновено е до няколко работни дни.
- Чести причини за отказ: липсваща политика за поверителност, грешен възрастов
  рейтинг, неработещ APK на чисто (без GMS) устройство.

## Бележка за GMS
Това приложение **не зависи от Google Play Services**. То е чист web + Capacitor
core и работи на устройства без Google услуги (типично за RUStore аудиторията).
