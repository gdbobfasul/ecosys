# Тестване — Монитор-робот (Huawei AppGallery)

## В браузър (най-бързо)
1. `npm install`
2. `npm run dev` → отвори http://localhost:5173
3. „Активирай робота" → разреши известия (браузърът ще попита) → Табло.
4. „+ Нов монитор" → натисни пресет (напр. Hacker News RSS) → „Тествай сега".
   - Ако видиш CORS грешка: това е НОРМАЛНО за някои източници в браузър.
     Постави безплатен CORS прокси в Табло → „CORS прокси" и тествай пак.
5. Добави монитора, включи главния ключ ON. Уеб таймерът проверява падежните
   монитори, докато табът е отворен.

## Симулиране на „ново съвпадение"
- Първата проверка само „заучава" текущите записи (без известие).
- При следваща проверка нови записи → известие + ред в дневника със зелен цвят.
- За JSON API като Hacker News Algolia новите истории се появяват бързо.

## На устройство (Android, изисква Android SDK — НЕ е налично в тази среда)
1. `npm run build`
2. `npx cap add android`
3. `npx cap sync`
4. `npx cap open android` → стартирай от Android Studio на емулатор/устройство.
5. Разреши „Известия" при подкана. Локалните известия се показват през
   `@capacitor/local-notifications`.

## Фонов режим (важно)
Уеб таймерът и `tick()` при отваряне покриват „проверка при отваряне" и докато
приложението е отворено. За ИСТИНСКИ периодичен фон (приложението затворено) е нужен
background-runner плъгин (напр. `@capacitor/background-runner`) + системни разрешения.
Това е документирано, не симулирано. Виж README → „Фонов режим".
